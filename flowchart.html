5. iLQR vs AL-iLQR 对比

维度 🟦 iLQR 🟥 AL-iLQR
求解问题 无约束轨迹优化 带不等式约束的轨迹优化
(控制量 u 的上下界)
算法结构 单层循环:
BackwardPass → LineSearch → ForwardPass		 双层循环:
外层: 更新 μ, σ (增广拉格朗日)
内层: 带约束梯度的 iLQR
BackwardPass Riccati 逆向扫描:
Quu = dldudu + dfduᵀ·Vxx·dfdu		 Riccati + 约束梯度项:
Quu = dldudu + dfduᵀ·Vxx·dfdu + didudu
Qu = dldu + dfduᵀ·Vx + didu
代价函数 J = Σ StateCost + TerminalCost
跟踪误差二次型		 J = Σ (StateCost + AL_Penalty) + TerminalCost
AL_Penalty = ½σ·‖max(μ/σ+g(u), 0)‖²
约束处理方式 ❌ 不处理约束
控制量可能超出界限		 ✅ 增广拉格朗日法
通过惩罚 + 乘子逐步逼近约束
收敛条件 代价变化 |ΔJ|/J < J_tolerance 约束违反度 ‖g(u)‖∞ < constraint_tolerance
超参数 reg, max_iter, β₁, β₂, γ, J_tol 同上 + al_max_iter, σ_factor, σ_max, constraint_tol
适用场景 没有严格控制量限制的平滑问题 有明确控制量约束的场景
(如 |δ| ≤ 0.5 rad, |a| ≤ 1.0 m/s²)
